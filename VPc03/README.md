# Спортивный клуб – база данных

## Таблицы
- **members** – участники (имя, email, телефон, дата рождения)
- **trainers** – тренеры (имя, специализация, контакты)
- **classes** – типы занятий (название, тренер по умолчанию)
- **class_sessions** – расписание (время начала/конца, вместимость)
- **memberships** – история абонементов (тип, срок действия)
- **payments** – проведённые оплаты
- **visits** – отметки о посещении конкретных сессий

## Ограничения
- Возраст участников проверяется на уровне приложения (в SQLite ограничение CHECK с julianday недопустимо)
- Вместимость сессии > 0 (`CHECK`)
- email участников – уникальный
- Все обязательные поля – `NOT NULL`

## Индексы (ускорение запросов)
- Индексы на все внешние ключи (`member_id`, `session_id`, `trainer_id`, `class_id`)
- Индекс на `start_time` для быстрого поиска пересечений
- Уникальный индекс автоматически создаётся для `email`

## Проверочные запросы (8 штук)
1. Активные абонементы на сегодня
2. Количество записанных на занятия по датам
3. Тренеры и их классы
4. Участники, не посещавшие клуб более 30 дней
5. Топ-3 популярных занятий по количеству посещений
6. Доход клуба по типам абонементов
7. Конфликты расписания (пересечения сессий одного участника)
8. Самый загруженный тренер по количеству проведённых сессий

## Как запустить (4 шага)
1. Установите Python 3.8+, зависимости: `pip install -r requirements.txt`
2. Выполните `python main.py` – создастся `sports.db`, схема, данные, запросы.
3. Откройте `sports.db` в DBeaver (или любой SQLite‑клиент).
4. Для проверки запросов скопируйте содержимое `checks.sql` в SQL‑редактор DBeaver и выполните.
