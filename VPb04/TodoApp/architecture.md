# Архитектура системы управления задачами (Todo-лист)

## Классы

### 1. `Task` (Задача)
**Назначение**: Представляет отдельную задачу.

**Атрибуты**:
- `title` : str — название задачи.
- `description` : str — описание.
- `is_completed` : bool — статус выполнения (по умолчанию False).
- `due_date` : Optional[date] — срок выполнения (может быть не задан).
- `created_at` : datetime — дата и время создания.
- `task_id` : UUID — уникальный идентификатор.

**Методы**:
- `mark_completed()` — отметить задачу выполненной.
- `mark_incomplete()` — снять отметку выполнения.
- `update_description(new_description: str)` — изменить описание.
- `set_due_date(date_str: str)` — установить / изменить срок.
- `to_dict()` — сериализация в словарь для сохранения.
- `__str__()` — строковое представление.

### 2. `Project` (Проект / Список задач)
**Назначение**: Контейнер для группы задач (например, "Работа", "Личное").

**Атрибуты**:
- `name` : str — название проекта.
- `tasks` : List[Task] — список задач.
- `project_id` : UUID — уникальный идентификатор.

**Методы**:
- `add_task(task: Task)` — добавить задачу в проект.
- `remove_task(task_id: UUID)` — удалить задачу по ID.
- `get_task(task_id: UUID)` — получить задачу по ID.
- `get_incomplete_tasks()` — вернуть список незавершённых задач.
- `get_completed_tasks()` — вернуть список завершённых задач.
- `to_dict()` — сериализация.
- `__str__()` — строковое представление.

### 3. `User` (Пользователь)
**Назначение**: Владелец проектов и задач.

**Атрибуты**:
- `username` : str — имя пользователя.
- `email` : str — электронная почта.
- `projects` : List[Project] — проекты пользователя.
- `user_id` : UUID — уникальный идентификатор.

**Методы**:
- `create_project(name: str)` — создать новый проект.
- `delete_project(project_id: UUID)` — удалить проект.
- `get_project(project_id: UUID)` — найти проект по ID.
- `to_dict()` — сериализация.

## Взаимодействие объектов
- `User` содержит несколько `Project`.
- `Project` содержит несколько `Task`.
- Методы пользователя управляют проектами, методы проекта управляют задачами.
- При сериализации в словарь обеспечивается возможность сохранения данных в JSON (реализовано в коде через вспомогательные функции).
