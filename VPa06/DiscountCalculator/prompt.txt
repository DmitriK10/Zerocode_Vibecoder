Ты — опытный Python-разработчик, строго следующий принципам SOLID.  
Твоя задача — написать CLI-приложение на Python для расчёта скидок.  
Необходимо предоставить **полные исходные коды всех файлов** и **подробное пошаговое объяснение** реализации.

## Техническое задание

### Общее описание
Разработать консольное приложение, которое:
1. Запрашивает у пользователя исходную цену товара.
2. Запрашивает процент скидки (целое число от 0 до 100).
3. Запрашивает фиксированную скидку по промокоду (необязательно, по умолчанию 0).
4. Рассчитывает финальную цену по формуле:  
   `final_price = max(0, price * (1 - discount_percent / 100) - fixed_discount)`.
5. Выводит результат расчёта.
6. Сохраняет каждый успешный расчёт в SQLite базу данных `calculations.db` с полями:  
   - `id` (автоинкремент)  
   - `original_price` (REAL)  
   - `discount_percent` (INTEGER)  
   - `fixed_discount` (REAL)  
   - `final_price` (REAL)  
   - `timestamp` (DATETIME, автоматически текущее время)
7. Предлагает пользователю выбор: выполнить новый расчёт, показать историю расчётов или выйти.

### Архитектурные требования (SOLID)
- **S**ingle Responsibility: каждый класс отвечает только за одну задачу.
- **O**pen/Closed: возможность расширения функционала (например, добавление новых типов скидок) без изменения существующего кода.
- **L**iskov Substitution: использовать абстрактные базовые классы и интерфейсы, где уместно.
- **I**nterface Segregation: интерфейсы должны быть узкоспециализированными.
- **D**ependency Inversion: зависимости от абстракций, а не от конкретных реализаций.

- Python 3.10+ (использовать современный синтаксис: `dataclasses`, аннотации типов).
- Для работы с SQLite использовать встроенный модуль `sqlite3`.
- Все функции и методы должны иметь аннотации типов.
- Код должен быть задокументирован docstring-ами на русском или английском языке.
- Соблюдать PEP 8.

- Обработка ошибок: неверный ввод (отрицательная цена, нечисловые значения) должна приводить к повторному запросу.
